# types

## 目的
提供项目通用类型与 Arena 领域类型，作为前后端/模块间契约的基础。

## 模块概述
- **职责:** 通用响应/分页/用户类型；Arena 领域类型（答案、引用、投票等）
- **状态:** 🚧开发中
- **最后更新:** 2025-12-24

## 规范

### 需求: 类型作为契约
**模块:** types
服务层与状态层应优先复用 `src/types/*`，避免重复定义与结构漂移。

#### 场景: 服务响应变更
当服务响应结构调整时，先更新 types，再同步更新服务与 UI。
- types 与实现保持一致
- 文档同步更新

### 需求: 衍生类型补齐
**模块:** types
当第三方库未导出所需类型时，可在 `src/types/` 中补齐衍生类型，以保证类型安全与复用。

#### 场景: Ant Design X 衍生类型
补齐 `@ant-design/x` 未导出的 `SourcesItem` 类型，供引用来源面板使用。
- 类型定义与库实现保持一致
- 仅补齐必要字段
